Measuring Traffic with Google Maps API

Spring 2017

Introduction

Poor commute is a common experience in large cities

As drivers we want to understand the general flow of traffic

But traffic sensor data is inaccessible to the public

Is there a simple way to analyze and discover traffic patterns by ourselves?

Idea #1: Google Maps

Step 1: Open map of Austin in browser

Step 2: Take a screenshot at set intervals using auto screenshot capture + auto refresh applications

Step 3: Count the number of green, orange, red, and maroon pixels using Python script

Step 4: Generate a traffic score

But...

Issues

Cannot use PC while running setup

Saving images and processing them one by one are inefficient

Inconsistencies syncing up all applications

Manual setup sometimes leads to human error

Idea #2

Google Maps Distance Matrix API

Automatically collect travel time from various routes

Easy setup: Set timer, start script, let it run in the background, data is written in csv file (instead of as large png files)

Convert time into traffic severity score

Aggregate scores

Results

Austin Traffic Score App

Simple web app is created to display general traffic patterns in Austin, divided by main regions

Uses Google Maps API + collected historical data to "predict" future traffic severity of a region

For personal use: See traffic severity of own commute routes

Limitations

Data needs to be validated with actual sensor data

Route selection and region mapping bias

Only two weeks' worth of data collected

Conclusion

Potential for method to be used in future traffic studies

Google Maps API: Inexpensive, accessible, and scalable

Useful on personal level: Data can inform user on traffic patterns within own commute
